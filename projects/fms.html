전세버스 ERP (FMS)

원더모빌리티 그룹 3개 법인 통합 차량관리시스템

Live Demo (Readonly)

Overview

전세버스 업체는 ERP 도입이 어렵습니다. 업무 프로세스가 체계화되지 않아 "무엇이 필요한지"조차 설명하기 어렵기 때문입니다.

이 프로젝트는 현업 담당자 인터뷰, 회의 녹음 분석, 18개 엑셀 파일 역분석을 통해 요구사항을 직접 도출하고, 4주 만에 프로덕션 수준의 ERP를 완성했습니다.

실시간 데이터 입력, 엑셀 임포트/익스포트, 리포트 생성 등 완전한 CRUD 기능이 구현되어 있으며, 포트폴리오용 데모는 데이터 보호를 위해 읽기 전용(Readonly)으로 제공됩니다.

개발 프로세스 (4주)

Week 1

현업 분석

  • • 담당자 인터뷰 및 회의 녹음 분석
  • • 18개 엑셀 파일 구조 역분석
  • • 요구사항 도출 및 우선순위 설정
Week 2

설계

  • • DB 스키마 설계 (25+ 테이블, 818 lines)
  • • 46개 거래처명 퍼지 매칭 테이블
  • • API 엔드포인트 설계 (30+ routes)
Week 3

핵심 개발

  • • Next.js 14 App Router 구조
  • • Prisma ORM + PostgreSQL (Supabase)
  • • OAuth 2.0 (Google, Kakao) + 승인 워크플로우
  • • 다중 엑셀 ETL 파이프라인
Week 4

완성 및 배포

  • • 16+ 페이지 UI 구현
  • • E2E 테스트 (Playwright)
  • • Vercel 배포 (readonly 모드)
  • • 문서화
